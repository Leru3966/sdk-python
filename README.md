## 🧭 First-Time Setup

After you download and open the app:

1. Create or edit a YAML file.
2. Add the agent steps you want to run.
3. Save the file in the same folder as the app, or in a folder you can find easily.
4. Start sdk-python.
5. Point the app to your YAML file when asked.

A simple setup may include:

- A task name
- One or more agent steps
- Input text
- Output settings
- Optional model choices

## 📄 Example YAML Workflow

A YAML file tells the app what to do. A basic workflow may look like this:

- Read a prompt
- Send it to an agent
- Pass the result to the next step
- Save the final output

This makes it easier to build multi-agent flows without writing a full program.

## 🤖 What sdk-python Can Do

sdk-python is built for YAML-driven agent orchestration. In plain terms, it lets you describe a workflow in a file and then run it.

Common uses include:

- Running one agent or many agents in order
- Sending tasks between steps
- Connecting to local or cloud-based models
- Using MCP-style tools and services
- Working with strands-agents based flows
- Testing ideas without building a full app

## 🧩 Common Workflow Parts

Many YAML files for sdk-python may use these parts:

- **agents**: the parts that do the work
- **inputs**: the text or data you give the workflow
- **outputs**: where results go
- **models**: the AI model to use
- **tools**: extra actions the agent can call
- **steps**: the order of the workflow

## 🧠 Tips for Best Results

- Keep your YAML file simple at first
- Use short step names
- Test one agent before adding more
- Save backups of working files
- Use clear input text
- Check spacing in YAML files, since spaces matter

## 🔧 Troubleshooting

### The app does not open
- Check that the file finished downloading
- Make sure you extracted the zip file if you downloaded one
- Try right-clicking the app and selecting **Run as administrator**

### Windows blocks the file
- Open the file details and check the publisher if shown
- If the file came from the release page, choose the option to run it

### The workflow does not start
- Open your YAML file and check the spacing
- Make sure each step has a name
- Check that the input file path is correct

### The output looks wrong
- Review the model setting
- Simplify the workflow
- Try one step at a time
